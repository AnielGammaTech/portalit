LootIT Reconciliation Reminders & Visibility

Three features built on top of the existing staleness system

Architecture Overview

How the 3 features connect
Feature 1
Sign-Off Reminders
Trigger: 30 days since last sign-off per customer

• Badge on global dashboard per customer
• Banner on customer detail page
• Telegram notification when due
Feature 2
Force-Match Alerts
Trigger: Follows sign-off expiry (same 30-day cycle)

• Orange badge on force-matched tiles
• Count in reconciliation filter tabs
• Included in Telegram reminder
Feature 3
Exclusion Visibility
Trigger: Exclusion age > 90 days without re-verification

• Badge on tiles with stale exclusions
• Shows excluded count + last verified date
• Flagged during sign-off verification
Shared Infrastructure
Expand useStalenessData
Add sign-off expiry + exclusion staleness to existing hook
Supabase Edge Function
Daily cron checks all customers, sends Telegram for due items
New DB columns
exclusion_verified_at on reconciliation_reviews, reminder_sent_at on sign_offs
